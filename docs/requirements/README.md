# Requirements

## Purpose
Store business requirements as the source of truth for downstream artifacts.

## Naming
- one feature per file
- file name: `<feature-name>.md` (lowercase kebab-case)

## Feature Name Examples
- `user-authentication.md`
- `role-permission-management.md`
- `session-lifecycle.md`

## Sub-Feature Decomposition
- If a parent file (example: `main-feature.md`) contains multiple sub-features, create one requirement file per sub-feature.
- Example:
  - `login.md`
  - `membership.md`
